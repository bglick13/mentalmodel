---
title: "Authoring Workflows"
description: "Structure programs as factory functions using Workflow, children, and Ref to express data dependencies."
---

A `Workflow` is the top-level container for every mentalmodel program. You assemble nodes into a tree of children and connect them with `Ref` to express data flow. The entire program lives in a plain Python function so it is easy to parameterize, test, and version.

## Workflow structure

`Workflow` accepts three key fields:

| Field | Type | Purpose |
|---|---|---|
| `name` | `str` | Stable identifier — becomes the graph ID in the IR |
| `children` | `Sequence[ChildT]` | Ordered list of child primitives |
| `description` | `str \| None` | Human-readable summary stored as IR metadata |

```python
from mentalmodel import Workflow, Actor, Effect, Ref

workflow = Workflow(
    name="my_pipeline",
    description="A short description of what this program does.",
    children=[
        Actor("data_source", handler=...),
        Effect("transform", handler=..., inputs=[Ref("data_source")]),
    ],
)
```

<Note>
`name` is the node's stable identifier throughout the IR, tracing, and metrics. Once you choose a name, treat it as permanent — renaming a node changes its graph ID and breaks stored run history.
</Note>

## Children define containment and local sequencing

The `children` list encodes both **containment** (which nodes belong to this scope) and **local sequencing** (the order in which sibling nodes are evaluated by the executor). Nodes earlier in the list are available as `Ref` targets for nodes that appear later.

You can nest containers — `Workflow`, `Parallel`, and `RuntimeContext` all accept `children` — to build hierarchical programs:

```python
Workflow(
    name="async_rl_demo",
    children=[
        RuntimeContext(
            name="local_control_plane",
            runtime="local",
            children=[
                Actor("policy_snapshot", handler=...),
                Actor("batch_source", handler=...),
                Effect("sample_policy", handler=..., inputs=[
                    Ref("batch_source"),
                    Ref("policy_snapshot"),
                ]),
            ],
        )
    ],
)
```

## Using Ref for cross-node dependencies

`Ref` declares a data dependency between two nodes. It contains a `target` (the name of the upstream node) and an optional `port` (defaults to `"default"`).

```python
from mentalmodel import Ref

# Reference the default output port of "policy_snapshot"
Ref("policy_snapshot")

# Reference a named port (advanced)
Ref("policy_snapshot", port="state")
```

You pass `Ref` objects in the `inputs` list of the consuming node. The executor resolves each `Ref` at runtime and passes the resolved values as a typed dict to the handler.

```python
Effect[SamplePolicyInputs, SamplePolicyOutput](
    "sample_policy",
    handler=PolicySampler(group_size=4, sampler_lag=0),
    inputs=[Ref("batch_source"), Ref("policy_snapshot")],
)
```

<Warning>
`Ref` targets must match the `name` of a node that is already reachable in the same scope or an enclosing scope. Referencing a node that does not exist causes a lowering error.
</Warning>

## Building the program as a factory function

Wrap every `Workflow` construction in a factory function that accepts configuration parameters and returns the fully-built `Workflow`. This pattern keeps your program pure (no global mutable state), easy to test with different configurations, and composable with the optimization layer.

```python
def build_program(
    group_size: int = 4,
    *,
    sampler_lag: int = 0,
    max_off_policy_steps: int = 0,
    initial_policy_version: int = 3,
) -> Workflow[AsyncRlNode]:
    """Build the async RL authoring model."""
    return Workflow(
        name="async_rl_demo",
        description="Milestone 2 async RL demo authored in semantic primitives.",
        children=[...],
    )
```

The return type annotation (`Workflow[AsyncRlNode]`) uses a `TypeAlias` that unions all legal child node types. This gives you static type-checking coverage across the entire program tree.

## Naming conventions

Follow these conventions for all node names:

- **Stable**: treat names as permanent identifiers, not display labels.
- **Lowercase snake_case**: `policy_snapshot`, `rollout_join`, `learner_update`.
- **Descriptive nouns or noun phrases**: the name should identify the role of the node, not its implementation.
- **No version suffixes**: `policy_snapshot_v2` is an antipattern — use the `description` field or metadata for version context.

## Full example: async_rl demo

The following is the complete factory function from the `async_rl` demo. It shows all the concepts above working together: nested `RuntimeContext` scopes, `Ref`-wired data flow, parallel fanout, a join reducer, an invariant, and incremental state.

```python
from mentalmodel import (
    Actor, Effect, Invariant, Join, Parallel, Ref, Workflow,
    extract_output_metrics, infer_output_metrics,
)
from mentalmodel.plugins.runtime_context import RuntimeContext


def build_program(
    group_size: int = 4,
    *,
    sampler_lag: int = 0,
    max_off_policy_steps: int = 0,
    initial_policy_version: int = 3,
) -> Workflow[AsyncRlNode]:
    return Workflow(
        name="async_rl_demo",
        description="Milestone 2 async RL demo authored in semantic primitives.",
        children=[
            RuntimeContext(
                name="local_control_plane",
                runtime="local",
                children=[
                    Actor[PolicySnapshotInputs, PolicySnapshotOutput, PolicySnapshotState](
                        "policy_snapshot",
                        handler=PolicySnapshot(initial_policy_version=initial_policy_version),
                    ),
                    Actor[BatchSourceInputs, list[PromptRecord], BatchState](
                        "batch_source",
                        handler=BatchSource(),
                    ),
                    RuntimeContext(
                        name="remote_sampling",
                        runtime="sandbox",
                        children=[
                            Effect[SamplePolicyInputs, SamplePolicyOutput](
                                "sample_policy",
                                handler=PolicySampler(
                                    group_size=group_size,
                                    sampler_lag=sampler_lag,
                                ),
                                inputs=[Ref("batch_source"), Ref("policy_snapshot")],
                            )
                        ],
                    ),
                    Parallel[...](
                        name="reward_fanout",
                        children=[
                            Effect[RewardInputs, RewardScores](
                                "pangram_reward",
                                handler=PangramScorer(),
                                inputs=[Ref("sample_policy")],
                            ),
                            Effect[RewardInputs, RewardScores](
                                "quality_reward",
                                handler=QualityScorer(),
                                inputs=[Ref("sample_policy")],
                            ),
                            Effect[RewardInputs, KLPrefetchOutput](
                                "kl_prefetch",
                                handler=KLPrefetch(),
                                inputs=[Ref("sample_policy")],
                            ),
                        ],
                    ),
                    Join[RolloutJoinInputs, RolloutJoinOutput](
                        "rollout_join",
                        inputs=[
                            Ref("policy_snapshot"),
                            Ref("sample_policy"),
                            Ref("pangram_reward"),
                            Ref("quality_reward"),
                            Ref("kl_prefetch"),
                        ],
                        reducer=RolloutJoinReducer(),
                    ),
                    Invariant[InvariantInputs, int](
                        "staleness_invariant",
                        checker=PolicyStalenessChecker(
                            max_off_policy_steps=max_off_policy_steps
                        ),
                        inputs=[Ref("rollout_join")],
                    ),
                    Actor[LearnerInputs, LearnerOutput, LearnerState](
                        "learner_update",
                        handler=LearnerUpdate(),
                        inputs=[Ref("rollout_join")],
                        metrics=[infer_output_metrics(prefix="mentalmodel.demo.learner_update")],
                    ),
                    Effect[RefreshInputs, RefreshOutput](
                        "refresh_sampler",
                        handler=RefreshSampler(),
                        inputs=[Ref("learner_update")],
                    ),
                ],
            )
        ],
    )
```

<Tip>
Run `build_program()` without arguments to get a zero-configuration program you can execute immediately during development and testing.
</Tip>
