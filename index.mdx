`mentalmodel` gives your Python programs a first-class semantic structure. Instead of ad hoc orchestration code, you describe your program using a small set of typed primitives such as `Workflow`, `Actor`, `Effect`, `Invariant`, `Parallel`, `Join`, and `Ref`, and the package turns that model into execution, observability, static analysis, and generated docs.

## Why mentalmodel?

Most programs grow opaque over time. Side effects scatter, invariants go unwritten, and execution order becomes implicit. `mentalmodel` makes every aspect of your program's structure explicit and machine-readable.

## Get started in 3 steps

<Steps>
  <Step title="Install">
    ```bash
    pip install mentalmodel
    ```
  </Step>
  <Step title="Author a workflow">
    ```python
    from mentalmodel import Workflow, Actor, Effect, Ref

    def build_program():
        return Workflow(
            name="my_pipeline",
            children=[
                Actor("fetch_data", handler=FetchData()),
                Effect("store_result", handler=StoreResult(), inputs=[Ref("fetch_data")]),
            ],
        )
    ```
  </Step>
  <Step title="Check and run">
    ```bash
    mentalmodel check --entrypoint mymodule:build_program
    mentalmodel verify --entrypoint mymodule:build_program
    ```
  </Step>
</Steps>
